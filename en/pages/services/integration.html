What is an MCP server?

MCP (Model Context Protocol) is an open standard through which AI agents talk to external systems, data and tools in a standardized way. Instead of building a custom one-off for every connection, there is one protocol – provider-independent and without vendor lock-in.

I build custom MCP servers that cleanly wrap your existing system (Microsoft 365, database, ERP, CRM or legacy) and make it securely accessible to your AI agents – read-only by default, with clearly limited permissions and logging. The MCP server is the access layer; what the agent then does with it (e.g. automating tasks) is the next step – see AI Automation.

Typical use cases

  • Find and summarize documents across SharePoint and Microsoft 365 sites
  • “What’s the order status / stock level?” – straight from SAP/ERP in natural language
  • Query CRM data (Salesforce/HubSpot) and update notes or status
  • Reporting via natural-language database queries (read-only)
  • Move data from PDFs/documents into the ERP in a structured way
  • Query and update project status in Jira

Most companies already have their data and tools – they’re just not reachable for AI. Copying, exporting and manual handovers don’t scale. Integration builds the secure bridge so AI can really work with your systems.

I rely on MCP as the standard and on custom code – a framework only when it brings a clear advantage. No no-code platforms: you keep control, your data stays where it belongs, and there’s no ongoing dependency on a third-party service.

Do I have to put my data in the cloud?

No. Access is read-only by default and the MCP server can run locally or on your own server – your data stays in your network.

Which systems can be connected – even old ones?

Yes. From Microsoft 365, databases, ERP and CRM to legacy systems via SOAP, FTP or CSV – and your own APIs.
